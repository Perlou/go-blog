# 博客文章写作提示词模版

## 使用说明

每次写新的博客文章时，只需要告诉 AI 以下信息，AI 就能帮你生成符合博客风格的文章。

---

## 快速模版（复制使用）

```
帮我写一篇博客文章：

【主题】：<文章主题，一句话概括>
【类型】：<技术教程/项目分享/经验总结/随笔/其他>
【要点】：
- <要点1>
- <要点2>
- <要点3>

【可选信息】：
- 面向读者：<初学者/中级开发者/高级开发者/通用>
- 包含代码：<是/否>
- 文章长度：<简短(800-1200字)/中等(1500-2500字)/详细(3000字+)>
- 特殊要求：<任何其他要求>
```

## 写作风格指南

### 1. 开篇风格

**技术文章**：直接切入主题，说明背景和意义

```markdown
随着 XXX 的广泛应用，如何有效 XXX 成为开发者面临的重要课题。
本文将分享...
```

**项目分享**：讲述项目背景和初衷

```markdown
这几天我基于 XXX 开发了一个...
这个项目从零到一的开发过程让我...
```

**随笔类**：营造氛围，引发思考

```markdown
窗外的阳光透过薄薄的云层洒进来...
最近总在思考...
```

### 2. 内容结构

**技术教程类**：

```markdown
## 背景/问题描述

## 解决方案/核心概念

## 实践步骤

### 步骤 1

### 步骤 2

## 最佳实践

## 注意事项

## 总结
```

**项目分享类**：

```markdown
## 项目初衷

## 技术选型

## 开发历程

### 阶段 1：...

### 阶段 2：...

## 核心功能展示

## 开发感受/反思

## 未来规划

## 总结
```

**经验总结类**：

```markdown
## 背景

## 问题/挑战

## 解决方案

### 方法 1

### 方法 2

## 效果对比

## 经验教训

## 总结
```

### 3. 写作要点

✅ **应该做的**：

- 使用二级标题（##）和三级标题（###）组织内容
- 代码块使用正确的语言标识（`python, `typescript 等）
- 适当使用引用、代码块、列表增强可读性
- 技术术语首次出现时可以简单解释
- 结尾要有总结，呼应开头
- 使用 emoji 适当点缀（🚀 ⚡ 🎨 💡 ✨ 等）

❌ **避免做的**：

- 标题过长或过于复杂
- 大段文字没有分段
- 代码块没有语言标识
- 缺少代码注释
- 结构混乱，没有层次

### 4. 代码示例规范

```python
# 代码应该：
# 1. 包含必要的注释
# 2. 使用清晰的变量名
# 3. 完整可运行（或标注伪代码）

def example_function(param: str) -> dict:
    """函数功能说明"""
    # 实现逻辑
    result = process(param)
    return result
```

### 5. 结尾风格

**技术文章**：总结要点，延伸阅读

```markdown
## 总结

本文介绍了...，通过...，我们可以...

记住：核心要点 1 + 核心要点 2 + 核心要点 3
```

**项目分享**：感受+呼吁

```markdown
## 总结

这次开发经历让我深刻感受到：

1. ...
2. ...

如果你对 XXX 感兴趣，欢迎访问：[链接]
欢迎 Star ⭐ 和提 Issue！
```

## 示例提示词

### 示例 1：技术教程

```
帮我写一篇博客文章：

【主题】：React 19 新特性实践指南
【类型】：技术教程
【要点】：
- React 19 的主要新特性（Server Components, Actions, use hook）

【可选信息】：
- 面向读者：中级 React 开发者
- 包含代码：是
- 文章长度：中等
```

### 示例 2：项目分享

```
帮我写一篇博客文章：

【主题】：开发了一个 Markdown 编辑器 Chrome 插件
【类型】：项目分享
【要点】：
- 项目背景（为什么做这个）
- 技术栈选择（Chrome Extension API + React + Monaco Editor）

- 开发中遇到的问题和解决方案
- Chrome 商店上架流程

【可选信息】：
- 包含代码：是（关键功能的代码片段）
- 文章长度：详细
- 特殊要求：重点讲解 Chrome Extension 开发的坑点
```

### 示例 3：经验总结

```
帮我写一篇博客文章：

【主题】：API 性能优化实战：从 2s 到 200ms
【类型】：经验总结
【要点】：
- 性能问题的发现和定位
- 优化方案（数据库索引、缓存、查询优化、异步处理）
- 优化前后的对比数据
- 经验教训

【可选信息】：
- 包含代码：是
- 文章长度：中等
- 技术栈：Node.js + PostgreSQL + Redis
```

### 示例 4：随笔

```
帮我写一篇博客文章：

【主题】：三年前端开发的一些思考
【类型】：随笔
【要点】：
```

## 封面图片

每篇文章都需要封面图。可以选择：

### 方案 1：使用 AI 生成（推荐）

```
同时帮我生成一张封面图，风格：<轻量化/渐变/扁平化/Material Design>
```

### 方案 2：使用现有图片

在 `/static/images/covers/` 目录中的现有图片，或从免费图库下载

### 方案 3：SVG 图标

如果是轻量级文章，可以使用 SVG 图标作为封面

## 文件命名规范

博客文章文件名格式：

```
YYYY-MM-DD-<url-friendly-title>.md
```

示例：

- `2025-11-21-react-19-new-features.md`
- `2025-11-21-my-markdown-editor-extension.md`
- `2025-11-21-api-performance-optimization.md`

---

## 补充说明

1. **日期格式**：自动使用当前时间，格式为 `YYYY-MM-DDThh:mm:ss+08:00`
2. **标签数量**：建议 3-5 个，保持精准
3. **文章长度**：根据主题复杂度调整，不要为了字数而凑内容
4. **代码质量**：确保代码示例是最佳实践，有必要的注释
5. **链接引用**：相关资源、项目链接使用 markdown 格式
