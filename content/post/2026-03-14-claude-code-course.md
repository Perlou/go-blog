最近这几年，AI 编码工具越来越多，GitHub Copilot、Cursor、Windsurf……但真正让我改变日常工作方式的，是 Anthropic 的 Claude Code。

用了几个月之后，我把学习过程整理成了一套开源课程，53 课时 + 12 个实战项目，从安装到 Agent SDK 全覆盖：

> 📚 **课程仓库**：[github.com/Perlou/claude-code-course](https://github.com/Perlou/claude-code-course)

下面聊聊我自己的使用感受，也算是这套课程的一个导读。

## Claude Code 是什么

一句话概括：它是一个跑在终端里的 AI 编码助手。

没有花哨的 GUI，没有插件依赖。在项目目录里敲 `claude` 进入交互模式，它就能读你的代码、改文件、跑命令、提交 Git。整个过程在终端完成，所有操作透明可回溯。

跟补全式的工具不同，Claude Code 是 **Agentic** 的——你给一个任务，它自己规划步骤、调用工具、读写文件，最后把结果交给你确认。你更像是一个 reviewer，不是逐行写代码的人。

这种 CLI-first + Agent 的组合，一开始不太习惯，但上手之后就回不去了。

## 日常怎么用的

说几个我用得最多的场景：

- 🔀 **Git 操作** — `claude commit` 自动生成提交信息，创建 PR 时自动写描述，这两个功能我每天都在用
- 🔧 **代码重构** — 给个明确指令，比如「把这个模块从 Class 迁移到 Hooks，接口保持不变」，多文件修改一次搞定
- 👀 **Code Review** — 把 diff 丢给它，能指出不少自己容易忽略的问题
- 📝 **写文档和测试** — 这类重复性高但需要理解上下文的活，交给它效率很高

日常开发大概 70% 的场景可以让 Claude Code 介入。不是说它每次都完美，而是它的参与省掉了大量机械性操作，让我能更专注在设计和决策上。

## 几个值得一提的能力

用了这么久，有几个功能我觉得特别值得单独说说。

**CLAUDE.md — 项目记忆**。在项目根目录放一个 CLAUDE.md，写上技术栈、代码规范、目录约定，Claude Code 每次启动时自动加载。相当于给它一份项目说明书，不用每次对话都重复交代背景。而且它支持多层级——全局、项目、目录、个人——不同项目可以有不同的规则。这个机制看起来简单，但对输出质量的提升是最明显的。

**MCP 服务器**。通过 MCP（Model Context Protocol），Claude Code 可以连接数据库、GitHub、Slack 等外部工具。这意味着它不止是个代码助手，还能串联你的整个开发工作流。

**子代理协作**。大任务可以拆给多个子代理并行处理，每个代理有独立的上下文和工具权限。配合 Git Worktree，多个代理甚至能在不同分支上同时干活。

**Skills 和 Hooks**。自定义技能（比如写一个 `/deploy` 一键部署）和生命周期钩子（比如每次写文件后自动跑 lint），让 Claude Code 可以嵌入你自己的工作流，变得真正可定制。

## 几条个人建议

这些都是我自己踩过坑之后的体会，不是什么金科玉律：

1. **先写好 CLAUDE.md 再开始用。** 没有项目说明的情况下，Claude 每次都要重新理解你的代码库。花半小时写一份，后面每次交互都更高效。

2. **给具体的指令，别给模糊的意图。** 「帮我优化一下」不如「把这个函数拆成三个，保持接口不变，加上类型注解」。约束越明确，结果越好。

3. **学会放手。** 给了指令就让它跑完，不要中途打断。结果不满意就给反馈让它改，比频繁介入效果好。

4. **养成 review 习惯。** 它做的每一步都是透明的，花几秒看一眼，既保障质量也能学到东西。

5. **迭代比完美重要。** 别纠结一次写出完美的 prompt，先拿到初步结果再调整，这样更快。

## 最后

Claude Code 改变了我写代码的方式——这话可能有点大，但确实是真实感受。它不是万能的，prompt 写得烂照样出烂结果，复杂的架构决策还得自己拿主意。但作为日常开发的搭档，它已经足够好了。
