提示工程（Prompt Engineering）是使用大语言模型时最关键的技能之一。本文将分享一些实用的提示工程技巧和最佳实践。

## 什么是提示工程

提示工程是设计和优化输入文本（提示词）的过程，目的是引导 AI 模型生成更准确、更有用的输出。好的提示可以显著提升模型表现。

## 核心原则

### 1. 明确性

提示应该清晰、具体，避免歧义。

**不好的提示**：

```

写一篇文章

```

**好的提示**：

```

写一篇 800 字的技术博客，主题是 Python 异步编程，
包含代码示例，面向有 2 年经验的开发者

```

### 2. 上下文信息

提供足够的背景信息帮助模型理解任务。

```

你是一位资深的前端工程师。请用 React Hooks 重写以下类组件：
[代码]

```

### 3. 示例驱动

使用 Few-shot Learning 提供示例。

```

将以下句子翻译成英文：

输入：今天天气很好
输出：The weather is nice today

输入：我喜欢编程
输出：I love programming

输入：这个项目很有挑战性
输出：

```

## 常见技巧

### Chain of Thought（思维链）

引导模型逐步推理：

```

问题：一个班级有 30 个学生，其中 60%是女生。
如果又转来 5 个男生，现在男生占比是多少？

请一步步分析：

1. 首先计算原来的男生和女生人数
2. 然后加上新转来的男生
3. 最后计算新的占比

```

### Role Playing（角色扮演）

让模型扮演特定角色：

```

你是一位经验丰富的代码审查员。
请审查以下 Python 代码，指出潜在问题和改进建议：
[代码]

```

### 约束条件

明确输出格式和限制：

```

用三个要点总结这篇文章，每个要点不超过 20 字：
[文章内容]

```

## 高级技巧

### 自我一致性

让模型生成多个答案并自我验证：

```

解决这个数学问题，给出 3 种不同的解法，
然后比较这些解法的结果是否一致。

```

### 迭代优化

根据输出反馈不断优化提示：

```

第一次尝试 → 分析问题 → 调整提示 → 第二次尝试 → ...

```

## 实际应用

### 代码生成

```

使用 TypeScript 编写一个通用的 API 请求封装函数，要求：

- 支持 GET、POST、PUT、DELETE 方法
- 自动处理错误和重试
- 支持请求拦截器
- 包含完整的类型定义
- 添加详细注释

```

```
为以下函数生成 JSDoc 文档：

- 包含参数说明
- 包含返回值说明
- 包含使用示例
- 包含可能抛出的异常

```

## 注意事项

1. **避免注入攻击**：验证用户输入，防止恶意提示
2. **成本控制**：tokens 数量会影响成本，优化提示长度
3. **隐私保护**：不要在提示中包含敏感信息
4. **持续学习**：模型在进化，提示技巧也要更新

## 总结

提示工程是一门艺术也是科学。通过不断实践和总结，我们可以更好地利用 AI 模型的能力，提升工作效率。

记住：好的提示 = 清晰的目标 + 充足的上下文 + 合理的约束
